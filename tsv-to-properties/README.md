# tsv-to-properties

TSV から Java の `.properties` ファイルを**環境ごと**に生成する。
10万〜100万行規模を想定し、1行ずつストリーム処理する。

## 使い方

```
node tsv-to-properties.js <入力TSV> <出力ルートDir> [options]
```

- `<入力TSV>` … 下記レイアウトの TSV ファイル
- `<出力ルートDir>` … `<出力ルートDir>/<env名>/<file>.properties` を生成する
- `--max-open-files <n>` … 同時に開くファイル数の上限（既定 200）
- `-q`, `--quiet` … 標準エラーへの完了サマリを抑制する
- `-h`, `--help` … 使い方を表示

```sh
node tools/tsv-to-properties/tsv-to-properties.js input.tsv ./out
# → ./out/PROD/messages.properties, ./out/STG/messages.properties, ...
```

データは `.properties` ファイル群として出力する（標準出力は使わない）。
完了サマリ（読込行数 / 環境数 / 生成ファイル数）を標準エラーへ1行出す（`-q` で抑制）。

```
tsv-to-properties: 読込 1000 行 → 3 環境 / 12 ファイル生成 → ./out
```

## 入力 TSV のレイアウト

タブ区切りで、1行目はヘッダ。

```
file    key    種別    <env名1>    <env名2>    ...    <env名N>    description
```

| 列          | 役割                                                             |
|-------------|------------------------------------------------------------------|
| `file`      | 出力プロパティファイル名。`<file>.properties` になる             |
| `key`       | プロパティのキー                                                 |
| `種別`      | 生成には**使わない**（読み飛ばす）                               |
| `<env名>`   | 環境ごとの値。**ヘッダの列名がそのまま出力ディレクトリ名**になる |
| `description` | キーの直前に付けるコメント（最終列）                           |

- 環境列は `種別` の次から `description` の手前まで。**数は可変**（1つ以上）。
- 環境名はヘッダから動的に読むので、実際の環境名をそのまま書けばよい。
- `file` 列は `config/app/messages` のようにサブディレクトリを含むパスでもよい。
  その場合 `<出力ルートDir>/<env名>/config/app/messages.properties` を生成し、
  中間ディレクトリは自動作成する。

### env セルの値とセンチネル

「空文字」と「キーが存在しない」を区別するため、env セルには次の3種を書く。
**空セルは禁止**（どちらか判別できないため、行番号付きエラーで停止する）。

| セルの値    | 出力              | 意味                                          |
|-------------|-------------------|-----------------------------------------------|
| 通常の値    | `key=値`          | その値                                        |
| `<EMPTY>`   | `key=`（空文字）  | 明示的に空。`getProperty` は `""` を返す       |
| `<NOKEY>`   | （行を出さない）  | この環境ではキーが未設定。`getProperty` は `null`|
| `<NOPROP>`  | （行を出さない）  | この環境ではプロパティ自体が適用外。`getProperty` は `null`|
| 空セル      | **エラー**        | —                                             |

- `<NOKEY>` と `<NOPROP>` は**出力挙動は同じ**（行を出さない）。
  TSV 上で「未設定なのか／そもそも適用外なのか」の**意図を区別するためのラベル**。
- `<EMPTY>` / `<NOKEY>` / `<NOPROP>` は実データに現れない前提のセンチネル。
  万一これらを正規の値として使いたい場合は破綻するので注意。

## 出力仕様

各行は、対象環境ごとに次の形で出力される。

```properties
#<description>
<key>=<value>
```

- **コメント**: `description` が空ならコメント行は出さない。
- **空値 / キーなし**: env セルのセンチネルで制御する（上の表を参照）。
  `<EMPTY>` は `key=`、`<NOKEY>`・`<NOPROP>` は行ごと省略、空セルはエラー。
  `<NOKEY>`・`<NOPROP>` の環境ではコメント行も出さない（孤立コメントを防ぐ）。
- **Unicode 変換**: `key` / `value` / `description` すべての非ASCII文字を `\uXXXX` に変換する
  （`native2ascii` 相当）。出力は純 ASCII になり、ISO-8859-1 として読む Java で文字化けしない。
- **エスケープ**: `java.util.Properties#store` と同じ規則。
  - キー: `=` `:` `#` `!` 空白 `\` をエスケープ（空白は全箇所）
  - 値: 同上だが空白は先頭のみエスケープ
  - コメント: 改行を空白に潰す

## 大規模ファイルでの挙動

- `readline` で1行ずつ処理するため、入力行数に対してメモリは一定。
- 出力ファイルは LRU プールで管理し、同時オープン数を上限（既定 200）に抑える。
  `file` の種類が多くてもファイルディスクリプタ枯渇を起こさない。
- 各出力ファイルは実行ごとに初回オープンで truncate（上書き）される。

## テスト

```sh
node --test tools/tsv-to-properties/tsv-to-properties.test.js
```

## 注意

- `file` 列の値はそのままファイル名になる。パス区切りや OS で禁止された文字が
  含まれる場合は、流用先で事前にサニタイズすること。
- TSV のフィールド内にタブを含むケースは想定しない（`description` 列のみ末尾結合で救済）。
