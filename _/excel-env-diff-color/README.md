# excel-env-diff-color

`file / key / 種別 / env1..envN / description` 形式の TSV を Excel に取り込み、
各行の **env 列を「値の種類ごと」に色分け** する VBA マクロ群。環境差分の違和感を目視で
あぶり出すための補助ツール。

## 公開マクロ

| マクロ              | 役割                                                   |
| ------------------- | ------------------------------------------------------ |
| `ImportTsvAndColor` | TSV を取り込み → 色分けまで一括（おすすめの入口）       |
| `ImportTsv`         | TSV を全列文字列・UTF-8 で新規ブックに取り込む（色分けなし） |
| `ColorEnvDiff`      | アクティブシートの env 列を色分けする                  |

## 前提とするレイアウト

| 列            | 役割                                           |
| ------------- | ---------------------------------------------- |
| file          | 任意                                           |
| key           | 任意                                           |
| 種別          | env 列に現れる値の種類数（**列の位置検出に使用**） |
| env1 … envN   | 環境ごとの値（種別の右隣から description の手前まで） |
| description   | 任意                                           |

- ヘッダーは **1 行目** に置く。
- env 列は「`種別` 列の右隣」から「`description` 列の手前」までを自動検出する。
  `description` 列が無ければ、`種別` の右側にある最終列までを env とみなす。
- env の数は 10 に固定していない（種別と description の間にある列数で決まる）。
- `種別` 列の値はマクロでは参照せず、**実際の env セルの値**から色分けを判定する。

## 配色ルール

各行ごとに独立して判定する。

1. env セルを値でグループ化する（**空セルも 1 つの値**として扱う）。
2. グループを「**出現数の多い順 → 同数なら登場 env が早い順**」で並べる。
3. 先頭グループ（＝最頻値＝基準）は **塗らない（白）**。
4. 残りのグループに、出現数の多い順でパレット色を割り当てる。

これにより、

- **1 セルだけ色が付く** → 1 環境だけ値が外れている
- **揃うはずの環境で色境界が崩れている** → 構造的な差分

が一目で分かる。種別 1（全環境同値）は塗られない。

### パレット（白＋最大 9 色）

| 順番  | 色      | RGB     |
| ----- | ------- | ------- |
| 基準  | 白      | 塗らない |
| 1     | 黄      | FFFF99  |
| 2     | 橙      | FFCC99  |
| 3     | 緑      | CCFFCC  |
| 4     | 水色    | CCFFFF  |
| 5     | ピンク  | FFCCFF  |
| 6     | 紫      | E0CCFF  |
| 7     | 赤      | FFCCCC  |
| 8     | 黄緑    | E6FF99  |
| 9     | グレー  | D9D9D9  |

env 10 列なら最大でも種別 10 = 白＋9 色で収まる。万一それを超える場合はパレットを循環させる。

## セットアップ

1. Excel で `Alt + F11` を押して VBE を開く。
2. メニュー「ファイル」→「ファイルのインポート」で `EnvDiffColor.bas` を取り込む。
   （または標準モジュールを追加し、`.bas` の中身を貼り付ける）

## 使い方

### A. ファイルから取り込む（おすすめ）

手で貼り付けると値が自動型変換され得るため、ファイルからの取り込みを推奨する。

1. `Alt + F8` →「`ImportTsvAndColor`」を実行する。
2. ダイアログで TSV ファイルを選ぶ。
3. 全列を**文字列・UTF-8** で新規ブックに展開し、続けて env 列を色分けする。

取り込みだけ行いたい場合は「`ImportTsv`」を実行する（色分けは別途 `ColorEnvDiff`）。

### B. すでに貼り付け済みのシートを色分けする

1. TSV を貼り付けた（または取り込んだ）シートをアクティブにする。
2. `Alt + F8` →「`ColorEnvDiff`」を実行する。
3. env 列が色分けされ、処理行数のサマリがダイアログで表示される。

再実行しても毎回 env セルを再設定するため、結果は冪等（前回の色が残らない）。

### 取り込み時の値の保全

`ImportTsv` / `ImportTsvAndColor` は全列を文字列(`xlTextFormat`)として開くため、
`01`・`1.0`・`2024-01`・`TRUE` などが数値・日付・真偽値に変換されない。
ファイルは UTF-8 前提（`Origin:=65001`）で読み込む。列数は先頭行のタブ数から判定する。

## 注意

- **色の意味は行ごとに独立**している。値の集合は行ごとに異なるため、
  行をまたいで同じ色でも別の値を指す。
- 種別が大きく多数派が無い行（全環境バラバラ）は、データ自体に基準が無い状態。
  色分けは「どの env がどの塊か」までは示すが、どれが正しいかは判断できない。
  この種の行は手動レビュー対象として扱う。
- 行数に比例した単純処理（O(行数)）。大量行では実行に時間がかかる場合がある。
